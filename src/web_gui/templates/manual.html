DNS Mars - User Manual

Version: 2.0 (Recursive Edition)

Last Updated: 2026-02-10

1. System Overview

DNS Mars is a high-performance DNS resolver designed for ISPs. It uses a Recursive Architecture, meaning it resolves domain names directly from root servers without relying on third-party upstreams like Google or Cloudflare. This ensures maximum privacy and independence.

2. Web Management GUI

Access the GUI at: https://YOUR_SERVER_IP:5000/

Key Features:

  • DNS Trust (Internet Positif): Toggle this switch to Enable/Disable blocking of regulated content.
    • Enable: Blocks gambling/porn domains (compliance mode).
    • Disable: Unblocks regulated content (for testing/special use).
    • Note: Malware blocking remains active regardless of this setting.
  • Flush Cache: Clears DNS cache immediately. Use this if a domain is not resolving correctly or has changed IP recently.
  • Restart Services: Safely restarts Dnsmasq and Unbound.

3. Self-Healing Guardian

The system runs an intelligent background service called Guardian.

What it does:
  • Checks DNS health every 10 seconds.
  • Automatically repairs broken services.
  • Updates firewall rules if the server IP changes.

4. Troubleshooting

High Disk Usage?

If the server disk fills up (logs can grow large):

  • Symptom: DNS stops working, Web GUI sluggish.
  • Fix: Logs are now automatically rotated. If critical, run:
sudo truncate -s 0 /var/log/dnsmasq.log
sudo systemctl restart dnsmasq

Client Cannot Browse?

  1. Check if the client's DNS IP is set to this server's IP.
  2. Try flushing the cache via the Web GUI.
  3. Check the "System Load" on the dashboard.

Google DNS Leaking?

The system is configured to block forwarding to 8.8.8.8. If you see Google DNS in a leak test:

  • Check your browser's "Secure DNS" (DoH) settings and disable them.
  • Ensure your router isn't forcing a secondary DNS.
